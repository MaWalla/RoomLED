Cheat Sheet

Keys are, whatever you've specified in the config for their respective devices. Values can be as follows:

  • 'individual' to assign an individual colour value for each led in the strip {'nodemcu1': {'individual': {'0': [r, g, b], '1': '#12df63'}}}

  • 'single_colour' to colour the entire strip in 1 colour e.g. {'nodemcu2': {'colour': '#abc'}} both hex and rgb lists work like: '#abcdef' or '#123' or [128, 32, 48]

  • 'random' well, it does what you'd expect. Each run presents a new colourful result! "{'nodemcu1': 'random'}"

  • 'gradient' input a list with 2 rgb values e.g "{'nodemcu3': {'gradient': [[12, 242, 55], [11, 9, 233]]}}" hex values can also be used: "{'nodemcu1': {'gradient': ['#ff2519', '#3eb']}}"

  • 'raveparty' Not implemented yet, since MaWalla was busy playing Minecraft... anyway: whoever is epileptical shouldn't get close to this e.g "{'nodemcu3': {'raveparty': 30}}" the number value is the delay between rgb flashes

You will need a config file with all devices, their type, ip address and LED count (and/or possibly some some other info)

An example could look like this:
{
"sideboard": {
"type": "nodemcu",
"ip": "192.168.5.100",
"leds": 105
},
"window": {
"type": "nodemcu",
"ip": "192.168.5.101",
"leds": 65
},
"computer": {
"type": "computer",
"ip": "192.168.5.10",
"devices": {
"arduino": {
"type": "1d",
"leds": 32
},
"blackwidow": {
"type": "2d",
"x": 22,
"y": 6
},
"launchpad": {
"type": "2d",
"x": 8,
"y": 8
}
}
},
"phone": {
"type": "phone",
"ip": "192.168.5.11"
}
}
Note that right now, only the NodeMCUs are supported.
